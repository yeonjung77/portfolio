Summary

  • 구축 대상 : DeepFashion-MultiModal 12,694장에 대해 픽셀 단위 파싱(24-class), 키포인트 검출(21 keypoints), 멀티 라벨 속성 분류(26-dim)를 단일 백본 위에서 3-Head로 동시 학습하는 멀티태스크 모델.
  • 3-Phase 학습 전략 : 1. SegFormer Parsing-only baseline → 2. SegFormer 멀티태스크 → 3. DeepLabV3+ 멀티태스크. 두 가지 핵심 비교 축(Single vs Multi-task / CNN vs Transformer)을 정량 분석
  • 최종 모델 선택 : SegFormer-B3 : 8개 metric 전부 SegFormer가 우위(mIoU −0.199, PCK −0.208, F1 −0.079), 학습 속도도 2.6배 빠름

문제 정의

패션 이미지에서 픽셀 단위 의류 영역 분할(파싱), 인체 관절 위치 추정(키포인트), 의류 속성 분류를 단일 모델로 동시에 수행한다.
DensePose annotation 미사용 : DeepFashion-MultiModal에 DensePose surface 좌표 맵도 제공되지만, 본 프로젝트의 메인 태스크인 24-class parsing이 이미 픽셀 단위 정답을 직접 제공하므로 입력은 RGB 3채널로 단순화했다.

1. 파싱 클래스 — 극심한 long-tail (background 77.37%)

  • 대용량 클래스(픽셀 > 2%): background, hat, hair, upper_clothes, skirt, right_arm, right_leg, face
  • 극소수 클래스(픽셀 < 0.05%): belt, bag, glove, socks, scarf, necklace, rompers, eyewear

→ mIoU는 24 클래스 평균이므로 소수 클래스가 학습 못 받으면 평균이 끌어내려진다. 실제로 학습 후 bag/socks/rompers/eyewear 4개 클래스가 정확히 IoU=0.0000으로 나타나 mIoU −0.167 견인했다.


2. 키포인트 visibility 분포 (93.65% visible)

→ 각 keypoint에는 visibility 값 0/1/2가 함께 라벨링되어 있고, 값에 따라 학습 loss와 평가 PCK에 어떻게 쓸지가 달라진다.

visibility의미학습 loss평가 PCK
0visible — 그대로 잘 보임✅ 사용✅ 평가
1occluded — 일부 가려졌지만 위치는 짐작 가능✅ 사용❌ 제외
2missing — 이미지 밖이라 좌표가 placeholder❌ 제외❌ 제외

가려진 점(occluded)은 좌표가 어디쯤일지 추정 가능하므로 학습 신호로는 쓰되, 평가에서는 제외했다.
이미지 밖(missing)은 좌표 자체가 의미 없어 학습·평가 모두에서 제외했다.

학습 방법

3-Head 멀티태스크 구조 (백본 공통)

Image [B, 3, 512, 384]
  → Backbone           → feat [B, C, h, w]   (SegFormer: C=512, DeepLab: C=2048)
  ├─ ParsingHead       Conv-BN-ReLU×2 + bilinear upsample → 1×1 Conv → [B, 24, H, W]
  ├─ KeypointHead      GAP → FC(C,512) → ReLU → Dropout(0.3) → FC(512, 63) → [B, 21, 3]
  └─ AttributeHead     GAP → Dropout(0.3) → FC(C,512) → ReLU → FC(512, 26)

Loss — Multi-task Weighted Sum

L_total = 1.0 × L_parse  +  0.1 × L_kp  +  0.5 × L_attr
  • Parsing — CrossEntropyLoss, pred/target spatial 다르면 F.interpolate
  • Keypoint — MSELoss(reduction='none') on (x, y), visibility==2 (missing)만 마스킹 제외
  • Attribute — BCEWithLogitsLoss

→ 멀티태스크 학습에서는 3개 loss를 더해 하나의 총 loss로 만든 뒤 한 번에 backward한다.

Loss다루는 값학습 초기 크기
Keypoint MSE좌표 (픽셀 단위, 0~500)~ 24,000
Parsing CE클래스 확률~ 0.3
Attribute BCE0/1 확률~ 0.7

keypoint loss가 다른 두 loss보다 약 3~8만 배 크므로 keypoint에 0.1을 곱해 다른 loss들과 비슷한 크기로 맞춰서 3개 task가 균형 있게 학습되도록 했다

전체 파싱된 픽셀 중 < 0.05%를 차지하는 소수 클래스 4개만으로 mIoU가 −0.167 떨어졌다 mIoU 0.54로 나온 직접적인 원인은 데이터 long-tail으로 파악했다. 같은 패턴으로 Attribute F1=0.40 미달의 원인또한 9개의 Fabric/Pattern 카테고리가 F1=0이 되어 평균을 크게 끌어내렸다.

2. 키포인트는 신체 부위에 따라 정확도 차이가 크다

부위예시 keypointPCK이유
상반신어깨 / 팔꿈치 / 코0.95+거의 항상 잘 보임
하반신 발끝toe0.62드레스·치마에 가려지는 경우 많음
귀ear0.7 근처머리카락에 가려짐

상반신과 하반신의 차이는 모델 한계가 아니라 입력 이미지에서 보이지 않고 가려진 데이터의 한계로 파악된다.

3. 다른 task를 같이 학습하면 메인 task(파싱) 성능이 떨어진다 — Negative transfer

같은 backbone에 같은 학습 조건으로 Single-task vs 3-tasks 결과표 :

Phase 2 첫 epoch에서 mIoU가 0.62 → 0.12로 급락 → 50 epoch 학습해도 0.62를 회복 못 했다. keypoint loss의 크기가 워낙 커 모델이 parsing보다 keypoint 쪽으로 학습 방향이 기울어졌다.

→ 여러 task가 같은 backbone을 두고 서로 다른 방향으로 학습을 끌어당기는 negative transfer(부정적 전이)가 발생했다.

최종 모델 선택 : SegFormer-B3 (Phase 2)

SegFormer 모델이 8개 metric 전부 우위 + 학습 속도 2.6배 + 모델 크기도 더 작다.
P1에서는 EfficientNet과 ViT가 사실상 동률이었던 것과 정반대로, P2에서는 픽셀 단위 예측을 여러 task로 동시에 푸는 환경에서는 백본 선택이 결과를 크게 좌우한다는 점을 정량적으로 확인했다. 특히 두 모델에서 PCK 격차가 가장 크게 나타난 것은, 이미지 전체를 한 번에 보는 구조인 Transformer가 CNN보다 keypoint 예측 같은 위치 정보에서 유리하다는 것을 확인시켜준다. 또한 소수 클래스에서는 두 모델의 차이가 없는 것으로, 모델 아키텍처보다는 데이터 자체의 한계가 더 결정적이라는 점도 알 수 있다.

엔지니어링 노트

  • 사전학습 가중치가 안 들어가는 함정 발견 & 해결. 초기 학습에서 mIoU가 0.43에서 멈추는 이상 증상이 보였다. 알고 보니 transformers 라이브러리가 기본으로 읽는 가중치 파일이 최근 보안 패치로 차단되어, 사전학습 없이 random 초기값으로 학습되고 있었다. use_safetensors=True 옵션을 사용했다.
  • 2단계로 나눠 학습해 안정성 확보. 처음부터 3-task를 동시 학습하지 않고, Phase 1에서 parsing 하나만 20 epoch 학습한 뒤 Phase 2에서 이 backbone을 이어받아 3-task를 학습해 Single-task와 Multi-task를 비교했다.
  • 3개 task를 균형 있게 보는 EarlyStopping 기준. 한 metric만 기준으로 학습 중단 시점을 정하면 나머지 두 task가 죽어버릴 수 있다. 그래서 0.4·mIoU + 0.4·PCK + 0.2·F1로 가중 평균한 값을 기준으로 사용했다.
