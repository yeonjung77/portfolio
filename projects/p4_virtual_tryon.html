Summary

  • 벤치마크 설계 : 공개 SOTA 가상 피팅 모델 두 가지를 동일 데이터·동일 쌍으로 추론해 어느 모델이 더 우수한지 정량 비교
  • 두 가지 평가 모드 : 원본 이미지의 의류를 지웠다가 다시 복원하는 paired(정답 있음 → SSIM·LPIPS)와 원본 이미지에 다른 의류를 입히는 unpaired(정답 없음 → FID)를 모두 측정해 충실도와 사실성을 함께 평가.
  • 결과 & 배포 : IDM-VTON이 세 지표 모두 우세함을 확인하고, 입력과 두 모델 결과를 비교하는 데모를 Gradio + HuggingFace Spaces로 배포.

문제 정의

StableVITON과 IDM-VTON은 이미 VITON-HD로 학습되어 공개된 모델이다. 따라서 동일 데이터로 재학습하는 것보다, 같은 조건에서 어느 모델이 더 우수한지를 공정하게 측정하고 재현 가능하게 만드는 것이 더 의미 있다고 판단했다. 본 프로젝트는 공식 코드와 체크포인트를 그대로 사용해 추론하고, 평가·시각화·데모는 구현하여 두 모델을 동일 선상에서 비교했다.

⚠️ 프로젝트에서 마주한 어려움

연구용 공개 코드를 실제로 돌려 비교하는 과정 자체가 핵심 난관이었다.

어려움구체적 상황
의존성 충돌두 모델의 라이브러리 버전 요구가 서로 달라, 한 환경에 같이 설치하면 깨졌다 → 모델별 conda 환경 분리
최신 GPU 비호환공식 안내 버전(torch cu117)이 실험 GPU(H100)에서 동작하지 않아, 호환되는 버전(cu118)으로 맞추는 디버깅 필요
체크포인트 손상공식 배포처(OneDrive)에서 받은 가중치 파일이 중간에 잘려 열리지 않음 → HuggingFace 미러에서 재확보
동시 비교 불가두 모델의 torch 버전이 달라 한 프로세스에 함께 올릴 수 없음 → 각각 추론해 결과를 저장한 뒤 비교
일반화 평가의 벽다른 데이터셋(DressCode)은 입력 형식(densepose)이 달라, 그대로는 공정한 일반화 비교가 불가능 (아래 인사이트 참조)

평가 방법

같은 모델을 두 가지 모드로 추론하고, 각 모드의 성격에 맞는 지표를 사용했다.

  • paired (원본 재구성) : 인물이 원래 입고 있던 의류를 그대로 복원하므로 원본 사진이라는 정답이 존재한다. 따라서 결과가 정답과 얼마나 같은지(유사도)를 재는 SSIM·LPIPS를 사용했다.
  • unpaired (교차 피팅) : 다른 의류를 입히므로 비교할 정답 이미지가 없다. 따라서 개별 비교 대신, 생성 결과 전체가 실제 사진 분포에 얼마나 가까운지(사실성)를 재는 FID를 사용했다.
지표모드의미방향
SSIMpaired원본과 구조적 유사도높을수록 ↑
LPIPSpaired원본과 지각적(딥피처) 거리낮을수록 ↓
FIDunpaired생성 결과 분포 vs 실제 사진 분포낮을수록 ↓

두 모델은 출력 해상도가 다르기 때문에(512×384 vs 768×1024) 동일한 기준으로 공정하게 비교하기 위해, SSIM·LPIPS는 두 결과를 같은 크기로 맞춰(resize) 계산했다. FID는 이미지를 한 장씩 비교하지 않고 결과 전체의 분포를 비교하는 지표로, 계산 과정에서 모든 이미지를 같은 크기로 변환하므로 해상도 차이의 영향을 받지 않는다.

인사이트

1. 성능 차이 원인 분석 - 해상도와 인코딩

특히 LPIPS에서 격차가 컸다 (0.079 vs 0.197, 약 2.5배). IDM-VTON은 높은 출력 해상도(768×1024)와 의류 외형·디테일을 함께 인코딩하는 구조 덕분에 질감·로고·주름을 더 잘 보존했다. StableVITON(512×384)은 더 가볍고 빠른 대신 디테일에서 뒤처졌다 — 품질과 속도의 트레이드오프로 해석된다.

2. DressCode 데이터셋을 활용한 일반화 평가 보류

두 모델이 학습한 VITON-HD는 in-domain 데이터다. 따라서 학습에 쓰이지 않은 다른 데이터셋(DressCode)에서도 성능이 유지되는지(일반화)를 추가로 확인하려 했다. (두 모델 모두 상의 try-on으로 학습됐기 때문에, 공정 비교를 위해 DressCode에서도 상의 카테고리만 사용할 계획이었다.)

그러나 densepose 입력 형식이 달라 보류했다. 두 모델은 VITON-HD 형식의 densepose로 학습되었는데, DressCode의 densepose를 표준 방식으로 변환해도 형식이 정확히 맞지 않았다.

형식이 어긋난 입력을 넣으면 점수가 불공정하게 낮아져 비교가 왜곡된다. 정확히 맞추려면 전처리 모델(detectron2)을 새로 돌려야 하나 비용 대비 효용이 낮다고 판단해 보류하고, 향후 과제로 남겨두었다.

엔지니어링 노트

  • 모델별 환경 격리 : 의존성 충돌을 피하기 위해 StableVITON·IDM-VTON·평가용 conda 환경을 분리하고, 총 8,128장(2모델 × 2모드 × 2,032쌍)을 안정적으로 추론했다.
  • 최신 GPU 대응 : StableVITON 추론 환경을 구성할 때, 공식 안내 버전(torch cu117)이 최신 GPU(H100)의 아키텍처를 지원하지 않아 추론 실행 시 CUDA 커널 오류가 발생했다 → 지원 버전(cu118)으로 교체. 또한 StableVITON의 공식 체크포인트가 배포처(OneDrive)에서 내려받는 중 손상돼, HuggingFace에 공개된 동일 가중치 사본(미러)에서 재확보했다.
