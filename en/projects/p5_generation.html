Summary

  • Domain fine-tuning: LoRA fine-tuned general-purpose Stable Diffusion v1.5 on FashionGen to turn fashion-specific text into product images.
  • Comparison experiments: Demonstrated the domain effect via before vs after fine-tuning, and analyzed the quality–cost trade-off via LoRA rank 4 vs 8.
  • Quantitative evaluation & deployment: Generated from 1,000 real captions to confirm FID improved 72.4 → 59.6, and deployed the results as a Gradio + HuggingFace Spaces demo.

Problem

General-purpose Stable Diffusion struggles with fashion-specific terms (e.g., jacquard, mandarin collar) and product-shot styling. Training a model from scratch is impractical, so the goal was to adapt off-the-shelf SD to the fashion domain by adding only a LoRA adapter and to verify the effect quantitatively.

⚠️ Challenges faced

Even with a proven library, running, comparing, and deploying involved several real-world issues to debug.

ChallengeSituation / Resolution
Library version conflictsLatest transformers (5.x) · numpy (2.x) · setuptools (82) broke with torch 2.1 → pinned compatible versions
Precision (dtype) conflictValidation generation crashed from mixing LoRA (fp32) + base (fp16) → wrapped in autocast
Evaluation protocolFID is meaningless on a few images → generated from 1,000 real captions for a distribution-level comparison
Data resolution limitSource data is 256px → upscaled to SD-native 512, and documented the limit honestly

Data & Method

DatasetFashionGen — curated heavy imbalance (2,972×) and duplicates into top-10 categories + 1 image per product (dedup) → 49,148 images (256 → 512 upscale)
CaptionsUsed input_description as-is (only 10% exceed CLIP's 77 tokens → auto-truncation)
ModelStable Diffusion v1.5 + LoRA (U-Net attention; VAE · Text Encoder · U-Net backbone frozen)
Trainingε-prediction MSE · AdamW 1e-4 · 15,000 steps · fp16 · single GPU (diffusers-based)

Evaluation

To compare under the same conditions as the real fashion distribution, 1,000 validation captions were used to generate images, then evaluated with two metrics.

Fine-tuning lowered FID from 72.4 to 59.6 (~16%), moving generated results closer to the real fashion distribution. Increasing rank to 8 gave nearly identical (slightly worse) results, confirming rank 4 is sufficient (saturated).

Insights

1. Domain effect — style consistency

Before fine-tuning, the base produced varied compositions (outdoor, lifestyle, etc.), whereas LoRA consistently generated a product-shot style with plain background, frontal view, and centered model. The qualitative change (product-shot style) matched the quantitative improvement (FID↓), confirming the domain adaptation worked.

2. Performance saturation by LoRA rank

Doubling LoRA rank from 4 to 8 yielded nearly identical FID/CLIP. In other words, performance already saturated at rank 4, so there was no benefit in increasing capacity. Rather than assuming excess capacity, the minimum sufficient capacity was identified through experiments.

3. Honest limitations

The absolute FID (~60) fell short of the target (≤30). This is because the data is 256px upscaled (limited detail) and the evaluation captions are noisy real product descriptions mixed with measurements and material codes. Rather than forcing a workaround, the cause was documented as a limitation, leaving higher-resolution data and caption refinement as future work.

Engineering Notes

  • Reproducible pipeline: Built an end-to-end flow — preprocessing (H5 → imagefolder) → LoRA training (diffusers-based) → bulk generation → FID·CLIP evaluation → demo — with a verification checkpoint at each stage.
  • Real-world troubleshooting: Diagnosed and resolved version conflicts (transformers/numpy/setuptools), an in-training dtype crash (autocast), the FID bulk-generation protocol, and large binary deployment (HF API upload).
