Summary

  • Scope : A multi-task model that simultaneously performs pixel-level parsing (24 classes), keypoint detection (21 points), and multi-label attribute classification (26-dim) on 12,694 DeepFashion-MultiModal images, learned with 3 heads on top of a single shared backbone.
  • 3-Phase strategy : 1. SegFormer Parsing-only baseline → 2. SegFormer multi-task → 3. DeepLabV3+ multi-task. Quantifies two key comparison axes: Single vs Multi-task / CNN vs Transformer.
  • Final choice : SegFormer-B3 — wins all 8 metrics over DeepLab (mIoU −0.199, PCK −0.208, F1 −0.079) and trains 2.6× faster.

Problem

From a fashion image, simultaneously perform pixel-level garment segmentation (parsing), human joint localization (keypoints), and garment attribute classification with a single model.
DensePose annotations were not used : DeepFashion-MultiModal bundles DensePose surface-coordinate maps, but the main task (24-class parsing) already provides per-pixel ground truth, so the input was kept simple at RGB 3-channel.

1. Parsing classes — extreme long-tail (background 77.37%)

  • High-volume classes (pixel > 2%): background, hat, hair, upper_clothes, skirt, right_arm, right_leg, face
  • Minority classes (pixel < 0.05%): belt, bag, glove, socks, scarf, necklace, rompers, eyewear

→ mIoU is averaged over 24 classes, so when minority classes fail to learn, the mean is dragged down. In practice, bag / socks / rompers / eyewear all ended up at exactly IoU=0.0000, dragging mIoU down by −0.167.


2. Keypoint visibility (93.65% visible)

→ Each keypoint is labeled with a visibility value of 0 / 1 / 2, which determines whether it contributes to the training loss and to the evaluation PCK.

visibilityMeaningTraining lossEvaluation PCK
0visible — clearly seen✅ used✅ evaluated
1occluded — partially hidden but position is inferable✅ used❌ excluded
2missing — outside the image, coordinate is a placeholder❌ excluded❌ excluded

Occluded points are still useful as training signal (the position can be guessed), but excluded from evaluation so only truly visible points contribute to the score.
Missing points carry no meaningful coordinate and are excluded from both training and evaluation.

EDA summary — findings and how they were handled

  • Data availability : Of 44,096 raw images, only 12.6K have precise parsing masks — used as the training pool with a 70/15/15 split.
  • Parsing class imbalance : Background alone occupies 77.37% of pixels; 16 of 24 classes are under 1% (max/min ratio 7,737×).
  • Keypoint visibility : Of 266,574 keypoints, visible 93.65% / occluded 5.76% / missing 0.59%. Lower-body ankle/toe show higher occluded ratios (dress/skirt effect).
  • Attribute long-tail : Among 26 dims, Pattern pure_color and Fabric cotton dominate; some Shape attributes (navel, ring) are very rare.
  • Annotation alignment : Image, parsing, and keypoint share the same coordinate frame, so augmentations must be applied identically to all three (when flipping horizontally, left/right labels must also be swapped, and mask resize uses NEAREST to preserve class IDs).

Training Method

3-Head multi-task architecture (shared backbone)

Image [B, 3, 512, 384]
  → Backbone           → feat [B, C, h, w]   (SegFormer: C=512, DeepLab: C=2048)
  ├─ ParsingHead       Conv-BN-ReLU×2 + bilinear upsample → 1×1 Conv → [B, 24, H, W]
  ├─ KeypointHead      GAP → FC(C,512) → ReLU → Dropout(0.3) → FC(512, 63) → [B, 21, 3]
  └─ AttributeHead     GAP → Dropout(0.3) → FC(C,512) → ReLU → FC(512, 26)

Loss — Multi-task Weighted Sum

L_total = 1.0 × L_parse  +  0.1 × L_kp  +  0.5 × L_attr
  • Parsing — CrossEntropyLoss; spatial mismatch handled via F.interpolate
  • Keypoint — MSELoss(reduction='none') on (x, y), masking only visibility==2 (missing)
  • Attribute — BCEWithLogitsLoss

→ In multi-task training, the 3 losses are summed into a single total loss and backpropagated together.

LossWhat it measuresEarly-training magnitude
Keypoint MSECoordinates (pixels, 0~500)~ 24,000
Parsing CEClass probabilities~ 0.3
Attribute BCE0/1 probabilities~ 0.7

Keypoint loss is roughly 30,000~80,000× larger than the other two, so it is multiplied by 0.1 to bring it down to a comparable scale — letting all 3 tasks train in balance.

Insights

1. Class imbalance is the ceiling for mIoU

ClassIoUEDA pixel ratio
bag / socks / rompers / eyewear0.0000 × 4 classes< 0.05%
gloves / left_shoe0.030.00~0.01%
scarf0.130.03%
background / skirt / left_arm0.99 / 0.87 / 0.8677% / 3.7% / 0.8%

Four minority classes — each occupying less than 0.05% of all parsed pixels — alone dragged mIoU down by −0.167. The direct cause of the mIoU 0.54 is the data long-tail. The same pattern explains the Attribute F1=0.40 shortfall: 9 Fabric/Pattern categories landed at F1=0, pulling the macro average down significantly.

2. Keypoint accuracy varies sharply by body region

RegionExample keypointPCKReason
Upper bodyshoulder / elbow / nose0.95+Almost always clearly visible
Lower body (toe)toe0.62Often hidden by dresses/skirts
Earsear~ 0.7Hidden by hair

The gap between upper and lower body is not a modeling limit but a data limit — what's hidden in the image cannot be learned.

3. Training all tasks together hurts the main task (parsing) — Negative transfer

Same backbone, same training setup — the comparison of Single-task vs 3-tasks:

From the very first epoch of Phase 2, mIoU collapsed from 0.62 down to 0.12, and even after 50 more epochs of training it never recovered to Phase 1's level of 0.62. The keypoint loss has such a large raw magnitude that the model's learning tilted toward keypoint and away from parsing.

→ When multiple tasks pull the shared backbone in different directions, the main task can take a hit — this is called negative transfer.

Final choice : SegFormer-B3 (Phase 2)

SegFormer wins on all 8 metrics, trains 2.6× faster, and is even smaller in size.
While P1 found EfficientNet and ViT effectively tied, P2 shows the opposite — when doing pixel-level prediction across multiple tasks at once, backbone choice strongly drives the result. In particular, the largest gap appearing in PCK reinforces that Transformer, which attends to the whole image at once, has an edge over CNN for spatial information like keypoint localization. On the other hand, the lack of any difference on minority classes shows that for those classes the data itself — not the architecture — is the deciding factor.

Engineering Notes

  • Caught and fixed a silent pretrained-weight loading bug. Early training showed mIoU plateauing at 0.43, which was suspicious. Turned out the default weight file path used by the transformers library was blocked by a recent security patch, so training was running with random weights instead of pretrained ones. Switching to use_safetensors=True resolved it.
  • Two-stage training for stability. Instead of training all 3 tasks together from scratch, Phase 1 trains parsing alone for 20 epochs, then Phase 2 resumes from this backbone and runs the full 3-task training — which also enables a direct Single-task vs Multi-task comparison.
  • EarlyStopping that balances 3 tasks. Stopping based on a single metric risks killing the other two tasks. So I use a weighted score, 0.4·mIoU + 0.4·PCK + 0.2·F1, as the stopping criterion.
