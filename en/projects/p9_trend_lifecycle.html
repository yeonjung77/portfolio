Fashion Trend Lifecycle Classification

A data-analysis project that uses STL time-series decomposition on Google Trends search data
to classify 156 fashion items into mega / micro / steady trends and infer their lifecycle stages and business strategies.

Summary

  • Scope : Classify 156 fashion items — each with Google Trends search interest (Korea · last 5 years · weekly) — into trend types and lifecycle stages, using only the shape of the search curve.
  • Pipeline : STL decomposition (trend + seasonality + residual) → 7 quantitative features → K-means (k=3) clustering → rule-based lifecycle stage. An unsupervised problem (no ground-truth labels) solved with clustering + rules.
  • Productizing the analysis : The trained model (scaler + KMeans) is serialized and deployed as an analytical tool that performs real-time trend classification for any new keyword.

Problem

A fashion trend demands a completely different buying/planning strategy depending on whether it is rising now, recurring every year, or a short-lived fad.
The goal is to extract quantitative features from the shape of the search-interest curve (uptrend · seasonality · volatility) and, without ground-truth labels, classify each item into mega · micro · steady and estimate its lifecycle stage.

Model Validation — Are k and the algorithm justified?

Since clustering is unsupervised (no labels), I validated "is k=3 right, is K-means best?" with the silhouette score rather than intuition. (Silhouette measures cohesion vs separation, −1 to 1.)

Raising k from 3→8 only nudges the silhouette 0.255→0.283, so there is no clear optimal k. HDBSCAN scores higher (0.322) but only by dropping 68 of 156 items (44%) as noise, which conflicts with the goal of classifying all items. → K-means (k=3) adopted (matches the business framing + no statistical gain from splitting further).

Insights

1. Trends are a continuous spectrum, not discrete categories

All cluster scores sit at 0.21–0.28 — the clusters don't separate cleanly. This isn't a failure: it reflects that many items straddle the mega↔steady boundary, the nature of trend data. So results are read as a soft tendency, not hard labels.

2. Honest classification without ground truth

Instead of supervised learning, I used clustering + rules and objectively compared algorithms and k via the silhouette score, making the selection rationale explicit. I prioritized "the choice that fits the goal (classify all items)" over "the highest score."

3. Productizing the analysis

Rather than a one-off study, the trained model (scaler + KMeans) is serialized and deployed as an analytical tool that classifies any new keyword in real time — turning exploratory analysis into a reusable artifact.

Limitations

  • Proxy for interest. Search interest is a proxy for attention, not actual sales.
  • Relative values. Google Trends is a 0–100 normalized relative value, so it's read by curve shape, not absolute comparison.
  • Feature-design dependence. The silhouette score assumes spherical clusters; results may shift with a different feature set or scaling.
