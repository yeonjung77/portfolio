Summary

  • Scope : A multi-task classifier predicting 6 attributes (category, subcategory, article type, color, season, usage) for 43,005 fashion product images with a single reproducible pipeline from EDA to inference.
  • Fair comparison : Trained EfficientNet-B3 and ViT-B/16 by varying only the backbone while keeping loss, optimizer, scheduler, augmentation, seed, and all other settings identical.
  • Key finding : The two models differ by only 0.0002 in avg_attr_f1 despite an 8× gap in backbone capacity — the performance ceiling stems from data quality and class imbalance, not the architecture.
  • Final choice : EfficientNet-B3 — equivalent accuracy at 1/8 the parameters, with meaningful gains in training and inference speed.

Problem

Build a model that predicts 6 attributes from 60×80 low-resolution fashion product images. Fashion search and recommendation services typically need to extract multiple attributes per product.

1. masterCategory — extreme imbalance (21,392×)

  • The top 4 categories (Apparel · Accessories · Footwear · Personal Care) account for 99.7% of the data.
  • For articleType, 73 of 143 classes have fewer than 50 samples, forming a long tail.

→ This justified the decision to remove articleType classes with fewer than 50 samples; as a result, masterCategory shrank from 7→5 and articleType from 143→68.


2. baseColour — 46 classes, top 5 colors cover 60%


3. Season · Gender · Usage — also skewed


EDA summary — findings and how they were handled

  • Missing values : only 0.78% (345 rows) → handled by simple removal.
  • CSV↔image mismatch : 5 rows in CSV without images, 22 images without CSV entries → unmatched rows dropped.
  • Resolution : all images fixed at 80×60 (small) → resized to 224×224 with ImageNet mean/std normalization for pretrained backbones. That said, the underlying 60×80 information is limited (subtle color tones are hard to distinguish).
  • Extreme class imbalance : masterCategory max/min ratio 21,397×, and all 6 targets are long-tailed → addressed with label_smoothing=0.1 and a stratified split (70/15/15) on masterCategory.
  • Long-tail color : 12 of 46 colors have fewer than 50 training samples → many minority classes are essentially untrainable (confirmed later by color F1=0.37).
  • Label ambiguity : pairs like Beige vs Cream, Off White vs White are hard even for humans → even a perfect model would still hit a metric ceiling due to label noise.

Training Method

6-Head multi-task architecture

Image [B, 3, 224, 224]
  → Backbone (timm)        → features [B, feature_dim]
  → Dropout(0.3)
  → heads["category"]      Linear → [B, 5]
  → heads["subcategory"]   Linear → [B, 30]
  → heads["articletype"]   Linear → [B, 68]
  → heads["color"]         Linear → [B, 46]
  → heads["season"]        Linear → [B, 4]
  → heads["usage"]         Linear → [B, 7]

Loss

total = 1.0 × L_category
      + 0.5 × mean(L_subcategory, L_articletype, L_color, L_season, L_usage)

All 6 heads use CrossEntropyLoss(label_smoothing=0.1).

Training settings (identical for both models)

OptimizerAdamW (backbone lr 1e-4, head lr 1e-3, weight_decay 0.01)
SchedulerCosineAnnealingLR (T_max=30, eta_min=1e-6)
Early Stoppingpatience 5, based on avg_attr_f1 (category saturates to 99% within 1~2 epochs)
Augmentation (train)RandomResizedCrop + Flip + ColorJitter + ImageNet Normalize
Augmentation (val/test)Resize 256 → CenterCrop 224
Shared hyperparamsbatch_size 32, epochs 30, image 224×224, seed 42

Only the backbone differs; everything else is held identical so any performance gap can be attributed purely to architecture.

Insights

Class imbalance directly determines per-class F1

The color scatter shows the pattern clearly: all 12 classes with fewer than 50 training samples converge to F1 ≈ 0, and the same pattern appears in usage (Party with 18 samples, Travel with 15). Performance is stable when training data is sufficient; when it isn't, no architectural change can compensate.

  1. EfficientNet vs ViT differ by only 0.0002 in avg_attr_f1. If backbone architecture were decisive, an 8× capacity increase (10.94M → 85.92M) would have produced a meaningful gap.
  2. color's weakness shows up identically on both backbones (0.37 vs 0.40) — the weakness can't be resolved by swapping architectures alone.

Confusion patterns also support label ambiguity

Model Comparison — CNN vs Transformer

Hypothesis for the small task-level gaps

  • EffNet wins on subcategory (+0.035) — CNN's local features are well suited to recognizing garment shape.
  • ViT wins on color (+0.030) and category_f1 (+0.020) — Transformer's global self-attention helps with global patterns like color tone and category.

Final choice : EfficientNet-B3

Equivalent accuracy at 1/8 the parameters, with advantages in both training and inference speed. Since ViT's extra capacity did not translate into better generalization on this dataset, EfficientNet is the more practical choice.

The generalization gap is nearly identical for both models. If the backbone affected robustness, the 8×-larger ViT would have shown either more overfitting or better generalization — neither materialized. The fluctuations come primarily from data, especially the distribution instability of minority-class tasks.

Engineering Notes

  • Single Source of Truth. 1_utils.TASK_KEYS defines the 6-task spec, and 2_preprocess.COLUMN_TO_TASK maps CSV columns to model keys. The model, loss, dataset, and trainer all reference these single sources, eliminating duplicated string literals and surfacing typos as immediate KeyErrors for faster debugging.
  • Modular pipeline. Responsibilities are split across files 1~7 (utils → preprocess → dataset → model → loss → scheduler → trainer) plus main.py, so each module can be tested independently.
